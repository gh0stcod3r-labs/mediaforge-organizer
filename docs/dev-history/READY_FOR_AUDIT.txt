## Key Design Decisions

1. **RenameEngine is the sole filesystem modifier**
   - UI can never touch files directly
   - All I/O goes through RenameEngine for safety & logging

2. **Plan then execute**
   - Generate preview (OperationPlan) before execution
   - Allows user confirmation

3. **Batch never aborts**
   - One file failure doesn't abort entire batch
   - Failures are logged and reported

4. **Undo from day 1**
   - Every successful operation logged
   - Can be undone (copy: remove, move: restore, rename: revert)

5. **Design system centralized**
   - constants.py holds all theming
   - Change one file to refresh entire UI
